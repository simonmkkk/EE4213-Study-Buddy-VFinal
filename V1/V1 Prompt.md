# V1 改進提示 - 6 個核心變更

## 專案目標與版本說明

### 目標
本文件記錄了基於可用性測試結果所需實施的 6 個核心改進。我們將創建兩個獨立的 GitHub 版本:

- **V1 版本 (Before)**: 可用性測試時使用的原始版本，包含所有發現的可用性問題
- **V2 版本 (After)**: 改進後的版本，實施了所有 6 個核心變更以解決使用者回饋的問題

### 工作流程
1. **當前狀態**: 代碼庫目前處於改進後的狀態 (V2)
2. **任務 1**: 根據 "Before" 描述還原代碼至 V1 版本，提交到 GitHub 作為 V1 發布
3. **任務 2**: 將代碼改回改進後的狀態 (V2)，提交到 GitHub 作為 V2 發布
4. **目的**: 清楚展示可用性測試前後的對比，用於專案報告和驗證

### 改進範圍
8 個核心改進涵蓋以下功能模塊:
- **海外交流模塊**: 變更 1, 2, 3 (視覺化學校探索器)
- **社群模塊**: 變更 4, 5 (評論功能)
- **專注學習模塊**: 變更 6 (計時器自訂)
- **工作資訊模塊**: 變更 7, 8 (職業歷程概覽與工作探索中心)

---

## 變更 1: 在所有需要返回的地方新增返回按鈕

### V1 版本 (Before - 可用性測試時的狀態):
- 視覺化學校探索器網格沒有導航欄
- 使用者選擇學校後卡在學校卡片上,沒有可見的返回路徑
- 其他深入頁面(如評論詳情、工作詳情等)也缺少明確的返回按鈕
- **問題:** 使用者在沒有明確返回上一層的方式時感到「迷失」,導致瘋狂的滑鼠移動,破壞了網頁瀏覽的心智模型。缺乏一致的返回導航降低了整體可用性

### V2 版本 (After - 改進後的狀態):
- 在所有可以返回的地方統一新增返回按鈕
- **海外交流模塊 (Overseas Exchange)**:
  - 視覺化學校探索器 (VisualSchoolExplorer): 選擇學校後在右上角顯示「返回學校」按鈕,返回學校列表
  - 課程自動配對器 (SyllabusAutoMatcher): 新增「返回」按鈕,返回視覺化學校探索器或海外交流主頁
- **社群模塊 (Community)**:
  - 學術牆 (AcademicWall): 選擇課程後新增「返回」按鈕,返回課程列表
  - 情緒中心 (EmotionCenter): 查看評論時新增「返回」按鈕,返回主動態
  - 保存的聊天 (KeptChats): 已有「返回」按鈕,返回靈魂配對 (✓ 已實施)
- **工作資訊模塊 (Job Information)**:
  - 已保存的工作 (SavedJobs): 新增「返回」按鈕,返回職業歷程概覽或工作資訊主頁
  - 已保存的資源 (SavedResources): 新增「返回」按鈕,返回職業歷程概覽或工作資訊主頁
  - 工作機會探索中心 (JobOpportunityHub): 查看工作詳情時新增「返回」按鈕,返回工作列表
  - 資源中心 (Resources): 查看資源詳情時新增「返回」按鈕,返回資源列表
  - 已申請的工作 (AppliedJobs): 新增「返回」按鈕,返回職業歷程概覽或工作資訊主頁
- **專注學習模塊 (Focus Learning)**:
  - 專注模式儀表板 (FocusModeDashboard): 進入專注會話後新增「返回」按鈕,返回儀表板
  - 微目標追蹤器 (MicroGoalTracker): 新增「返回」按鈕,返回專注學習主頁
- **設計規範**:
  - 所有返回按鈕使用一致的設計風格,帶有向左箭頭圖示 (ArrowLeft 組件)
  - 返回按鈕統一放置在頁面左上角或標題旁邊,保持視覺一致性
- **理由:** 在所有需要的地方增加一致的導航返回選項,大幅降低認知負荷,建立統一的導航模式,符合使用者對可逆操作的期望,提升整體應用程式的可用性和使用者體驗

## 變更 2: 在所有列表頁面實作自由文字搜尋功能

### V1 版本 (Before - 可用性測試時的狀態):
- 僅有國家和課程的下拉式篩選器
- 使用者嘗試輸入學校名稱但無法
- 沒有可見的搜尋欄
- **需要移除的已實施搜尋功能**:
  - 工作機會探索中心 (JobOpportunityHub): 已有搜尋欄,需移除
  - 資源中心 (Resources): 已有搜尋欄,需移除
  - 學術牆 (AcademicWall): 已有搜尋欄,需移除
- **問題:** 下拉式限制使尋求特定輸入的使用者感到挫折。使用者指出「那會快得多」,突顯了缺乏輸入靈活性。V1 版本應該只有基本的下拉式篩選器,沒有任何搜尋功能

### V2 版本 (After - 改進後的狀態):
- 在所有列表和探索頁面統一新增自由文字搜尋功能
- **海外交流模塊 (Overseas Exchange)**:
  - 視覺化學校探索器 (VisualSchoolExplorer): 在篩選器上方新增搜尋欄,可輸入國家、專業或學校名稱進行篩選
  - 課程自動配對器 (SyllabusAutoMatcher): 新增課程搜尋欄,可根據課程代碼或名稱搜尋
- **社群模塊 (Community)**:
  - 學術牆 (AcademicWall): 新增課程搜尋欄,可根據課程代碼或名稱搜尋 (重新實施)
  - 情緒中心 (EmotionCenter): 新增貼文搜尋欄,可根據標籤或內容關鍵字搜尋
  - 保存的聊天 (KeptChats): 新增聊天搜尋欄,可根據配對使用者名稱或主題搜尋
- **工作資訊模塊 (Job Information)**:
  - 工作機會探索中心 (JobOpportunityHub): 新增工作搜尋欄,可根據公司名稱、職位或技能標籤搜尋 (重新實施)
  - 資源中心 (Resources): 新增資源搜尋欄,可根據資源名稱、類別或標籤搜尋 (重新實施)
  - 已保存的工作 (SavedJobs): 新增搜尋欄,可快速篩選已保存的工作
  - 已保存的資源 (SavedResources): 新增搜尋欄,可快速篩選已保存的資源
  - 已申請的工作 (AppliedJobs): 新增搜尋欄,可根據公司或職位搜尋已申請的工作
- **專注學習模塊 (Focus Learning)**:
  - 微目標追蹤器 (MicroGoalTracker): 新增目標搜尋欄,可根據目標名稱或標籤搜尋
- **設計規範**:
  - 所有搜尋欄使用一致的設計和位置(頁面頂部篩選區域)
  - 即時搜尋,無需點擊搜尋按鈕
  - 支援部分匹配和不區分大小寫
  - 搜尋圖示 (Search icon) 提供視覺提示
  - 清晰的 placeholder 文字說明可搜尋內容
  - 下拉式選單仍然可用作為替代篩選選項
- **理由:** 自由文字搜尋增強了可發現性,並匹配使用者高效查詢的心智模型,大幅減少任務時間。統一在所有列表頁面實施搜尋功能,建立一致的使用者體驗

## 變更 3: 以清晰的文字標籤取代模糊的評論數量圖示

### 當前版本狀態:
- **檔案位置:** `src/pages/overseas-exchange/VisualSchoolExplorer.tsx`
- **當前實作:** 學校卡片上使用人物圖示(👤)加數字顯示評論數量
- **具體程式碼:**
  ```tsx
  <button className="flex items-center gap-1 text-sm text-muted-foreground">
    <span role="img" aria-label="people">👤</span>
    <span>{commentDisplayCount}</span>
  </button>
  ```

### V1 版本需要的操作 (Before - 可用性測試時的狀態):
**目標:** 保持模糊的圖示顯示,展示問題狀態

**操作步驟:**
1. **保持當前的人物圖示 (👤)**: 不做任何修改
2. **確保沒有文字標籤**: 只顯示圖示和數字
3. **移除任何工具提示或說明文字**: 讓使用者困惑「這是評論還是查看次數?」

**預期結果:**
- 學校卡片上只有模糊的人物圖示 + 數字
- 沒有任何文字說明或提示
- 使用者會感到困惑,不確定數字代表什麼意思

### V2 版本需要的操作 (After - 改進後的狀態):
**目標:** 使用清晰的文字標籤取代模糊圖示

**操作步驟:**
1. **替換人物圖示為對話框圖示**: 將 👤 emoji 改為 `<MessageSquare>` 圖示
2. **添加文字標籤**: 將 `{commentDisplayCount}` 改為 `{commentDisplayCount} 則評論`
3. **添加工具提示**: 使用 Tooltip 組件包裝,顯示「學生評論數量」
4. **改進程式碼:**
   ```tsx
   <Tooltip>
     <TooltipTrigger asChild>
       <button className="flex items-center gap-2 text-sm text-muted-foreground">
         <MessageSquare className="h-4 w-4" />
         <span>{commentDisplayCount} 則評論</span>
       </button>
     </TooltipTrigger>
     <TooltipContent>學生評論數量</TooltipContent>
   </Tooltip>
   ```

**設計指南:**
- 使用對話框圖示 (MessageSquare) 明確表示評論
- 文字標籤使用「X 則評論」格式
- 滑鼠懸停時顯示工具提示
- 保持與現有設計系統一致的樣式

**理由:** 文字標籤改善了語意清晰度和可及性,符合格式塔識別規則,提升了使用者的理解能力,消除了「這是評論還是查看次數?」的困惑

## 變更 4: 啟用評論編輯和刪除功能

### V1 版本 (Before - 可用性測試時的狀態):
- 使用者提交評論後無法編輯或刪除
- 沒有修正機制
- **問題:** 缺乏修正機制導致情緒壓抑和較低的滿意度,加劇了分享空間中的信任問題

### V2 版本 (After - 改進後的狀態):
- 使用者可以編輯已提交的評論
- 使用者可以刪除自己的評論
- 提供清楚的編輯和刪除按鈕
- **理由:** 刪除和編輯功能使使用者能夠隨時完善或移除情緒內容而不感到恐懼,增強信任並符合情緒需求,同時保持真實分享的乾淨空間

---

## 變更 5: 將社群評論移至專用頁面

### V1 版本 (Before - 可用性測試時的狀態):
- 評論直接顯示在主社群動態消息中(內聯顯示)
- 造成視覺雜亂
- 增加滾動疲勞
- **問題:** 內聯評論造成視覺雜亂並增加滾動疲勞,破壞了格式塔接近性,並以混合內容壓垮使用者。使用者表示「有太多文字了......我只想看其他貼文」

### V2 版本 (After - 改進後的狀態):
- 評論移至專用的獨立頁面
- 主動態消息只顯示貼文摘要和「查看評論」按鈕
- 點擊後進入專用評論頁面
- **理由:** 分離評論減少了資訊過載並改善了焦點,遵循分塊技術以簡化導航,允許使用者深入參與而不失去主要動態消息的脈絡

---

## 變更 6: 在專注學習中新增自訂專注時間的下拉式清單

### V1 版本 (Before - 可用性測試時的狀態):
- 固定的計時器選項(例如只有 25 分鐘)
- 使用者無法選擇自己想要的專注時間
- **問題:** 僵化的計時器選項限制了個人化,導致具有不同學習習慣的使用者感到挫折和脫離。使用者表示「我想要 45 分鐘,不只是 25 分鐘,我該如何更改?」

### V2 版本 (After - 改進後的狀態):
- 提供下拉式清單讓使用者選擇專注時間
- 包含多個時間選項(例如 15、25、45、60 分鐘)
- 使用者可以根據自己的需求選擇合適的時長
- **理由:** 靈活的選項符合使用者對可自訂工具的心智模型,減少僵化性,增強感知控制和課程有效性

---

## 變更 7: 標準化職業歷程概覽的進度條顏色並新增工具提示

### V1 版本 (Before - 可用性測試時的狀態):
- 進度條使用不一致的顏色(個人資料為藍色,成功為橙色)
- 指標未定義,沒有解釋(例如「成功」40%)
- 使用者對其狀態感到困惑
- **問題:** 未指定的顏色和未定義的指標違反了視覺層次原則,增加了認知負荷。使用者表示「為什麼個人資料是藍色,成功是橙色?40% 是不好的嗎?『成功』到底是什麼意思?」

### V2 版本 (After - 改進後的狀態):
- 標準化進度條顏色系統
- 為每個指標新增清晰的工具提示說明
- 提供進度百分比的上下文解釋
- **理由:** 標準化的視覺效果和解釋遵循視覺層次規則,澄清狀態以減少職業不確定性,提升動機和對工具的信任

---

## 變更 8: 在工作機會探索中心為工作標籤新增搜尋欄

### V1 版本 (Before - 可用性測試時的狀態):
- 顯示工作標籤(例如 AI、雲端、金融)但無法搜尋
- 標籤無法用於篩選特定興趣
- 缺少搜尋功能
- **問題:** 標籤在沒有搜尋功能的情況下創造了虛假的可供性,挫敗了使用者對可篩選探索的心智模型。使用者表示「沒有搜尋欄讓我搜尋特定的實習?」

### V2 版本 (After - 改進後的狀態):
- 新增標籤搜尋欄功能
- 使用者可以點擊標籤或輸入關鍵字進行篩選
- 支援多標籤組合搜尋
- **理由:** 啟用基於標籤的搜尋滿足使用者對靈活探索的意圖,減少挫折感,提升工作機會的可發現性

---

## 總結

這 8 個核心改進針對可用性測試中發現的關鍵問題:

### 改進對應表
1. **導航問題** → 變更 1: 新增返回按鈕
2. **搜尋靈活性** → 變更 2: 實作自由文字搜尋
3. **資訊清晰度** → 變更 3: 改用文字標籤
4. **使用者控制** → 變更 4: 啟用編輯/刪除功能
5. **資訊架構** → 變更 5: 分離評論至專頁
6. **個人化需求** → 變更 6: 提供時間自訂選項
7. **視覺一致性** → 變更 7: 標準化進度條顏色與指標說明
8. **搜尋功能** → 變更 8: 新增工作標籤搜尋欄

### 影響的功能模塊
- **海外交流 (Overseas Exchange)**: 變更 1, 2, 3
- **社群 (Community)**: 變更 4, 5
- **專注學習 (Focus Learning)**: 變更 6
- **工作資訊 (Job Information)**: 變更 7, 8

### 實施策略
所有改進都基於真實使用者測試回饋,旨在:
- 降低認知負荷
- 增強使用者控制感
- 改善整體使用者體驗
- 提升功能可用性與滿意度
